Smart Steps AI API Documentation

REST API for Smart Steps AI Professional Persona system, a tool for facilitators, therapists, probation officers, and similar professionals.

Swagger UI Documentation ReDoc Documentation

Introduction

The Smart Steps AI API provides programmatic access to the Smart Steps AI Professional Persona system. This API allows you to:

  • Create and manage AI professional personas with detailed backstories, expertise areas, and conversation styles
  • Start and manage therapy sessions
  • Exchange messages with AI personas
  • Access real-time conversation through WebSockets
  • Generate insights and analysis from conversation data

Base URL

https://api.smartsteps.ai/api/v1

All API requests should be prefixed with this base URL.

Authentication

The API uses JWT (JSON Web Token) for authentication. To authenticate:

  1. Make a POST request to /auth/token with your username and password
  2. Use the returned token in the Authorization header for subsequent requests

Example

POST /auth/token
Content-Type: application/x-www-form-urlencoded

username=your_username&password=your_password

Response:

{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}

Using the token:

GET /sessions
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Keep your tokens secure and never share them!

Key Endpoints

POST /auth/token

Authenticate and get an access token.

GET /personas

List all available professional personas.

POST /personas

Create a new professional persona.

GET /sessions

List all therapy sessions.

POST /sessions

Create a new therapy session.

POST /conversations/{session_id}/messages

Send a message in a therapy session and receive an AI response.

GET /conversations/{session_id}/messages

Get conversation history for a therapy session.

WS /ws/{session_id}

WebSocket endpoint for real-time conversation.

POST /analysis/sessions/{session_id}/analyze

Generate insights and analysis from a therapy session.

Response Codes

Code Description
200 Success
201 Created
204 No Content (successful deletion)
400 Bad Request (invalid input)
401 Unauthorized (missing or invalid token)
403 Forbidden (insufficient permissions)
404 Not Found
422 Validation Error
429 Too Many Requests (rate limit exceeded)
500 Internal Server Error

Rate Limiting

The API implements rate limiting to prevent abuse. If you exceed the rate limit, you will receive a 429 Too Many Requests response.

The current rate limit is 100 requests per minute per IP address.

Webhooks

The API can send webhooks for various events. To receive webhooks, you need to register a webhook URL in your account settings.

Webhook events include:

  • Session created
  • Session completed
  • New message received
  • Analysis completed

SDKs and Client Libraries

We provide official client libraries for several languages:

  • Python SDK
  • JavaScript SDK
  • C# SDK for Unity
